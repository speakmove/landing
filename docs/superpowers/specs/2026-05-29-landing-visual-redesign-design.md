# Визуальный редизайн лендинга SpeakMove — дизайн-спека

**Дата:** 2026-05-29
**Статус:** утверждено в брейншторме (визуальный компаньон), готово к написанию плана реализации
**Охват:** визуальная раскладка секций главной, /pricing, /how-it-works + глобальный ритм страницы, декоративные разделители и шапка. Контент/копи — без переписывания (только косметика, где указано).

## 0. Цель и принципы

Пересмотреть визуальное расположение блоков сайта на современные, небанальные решения (2025–2026), сохранив бренд (emerald + gold + cream, мягкие тени, скруглённые карточки, Telegram/voice-эстетика) и не ломая копи.

**Сквозные принципы:**
- **Единый «парящий» почерк** — наклонённые пузыри/пилюли с мягкой тенью повторяются в нескольких блоках.
- **Voice-ДНК** — волна голоса как фирменный мотив (hero-декор, FinalCta, статы).
- **Карта/панель = акцент, не дефолт** — большинство секций спокойные, выделяем только важное.
- **Анимации** — обязательный `prefers-reduced-motion` fallback для каждого эффекта.

### Ограничения проекта (обязательны, см. CLAUDE.md)
- **Нет inline `<svg>`** — иконки/волны/divider/donut — компоненты в `06_shared/ui/Icon/`, ре-экспорт из `@/shared/ui`, либо чистый CSS.
- **Нет arbitrary Tailwind-значений** — токены в `@theme` (`app/globals.css`).
- **Нет inline `style={{}}`** — всё через `className`; динамика — через CSS-переменные из `@theme`.
- **FSD** — слайсы одного слоя не импортируют друг друга; хуки в `model/hooks/useFoo.ts`; публичный API через `index.ts`.
- **Tailwind v4 layer trap** — CSS-оверрайды утилит вне `@layer components`.
- **CSP** — при новых доменах/шрифтах обновлять allowlist в `next.config.ts`.

## 1. Глобальный ритм страницы (весь сайт) — «Подход 2: полосы + акценты»

- **Большинство секций — полосы во всю ширину** с **чередованием фона** (белый / мягкий `surface`). Спокойный ритм в духе Stripe/Linear; контент — в центрированном `Container`. Это дефолт.
- **Приподнятая «карта-панель» (скругление `--radius-card-lg`, тень `--shadow-mid`) — ТОЛЬКО как акцент** для важных блоков (карточка тарифа). Не оборачивать каждую секцию в карту.
- **Full-bleed цветные секции** (край-в-край): тёмная «Основатели» (ink-navy `#0b1220`) и emerald «FinalCta». Только они «прорываются» на всю ширину.

### Декоративные разделители (волна)
- **Стиль: одиночная плавная волна (sine).** Shared-`Icon` (`WaveDivider`), **без inline svg**. `role="presentation"`, `aria-hidden`.
- **Правило цвета: заливка волны = цвет секции-приёмника (нижней), всегда.**
- **Волна — ТОЛЬКО на входе в full-bleed цветную секцию.** Между полосами с чередованием фона волны нет.
- **Точные места:**
  - **Главная:** (1) вход в тёмную «Основатели» (волна ink-navy); (2) вход в emerald «FinalCta» (волна emerald).
  - **/pricing:** одна — перед emerald «FinalCta».
  - **/how-it-works:** одна — перед emerald «FinalCta».

### Eyebrow/kicker/crumb — шрифт
Все eyebrow/kicker/crumb → `--font-mono` (`"JetBrains Mono", "JetBrains Mono Fallback", ui-monospace, Menlo, monospace`). Единообразно во всех секциях.

## 8. Анимации (библиотека + список) — утверждено

**Библиотека: `framer-motion` (уже в проекте, ^12.39.0). GSAP НЕ добавляем** — framer-motion + CSS-keyframes + `lenis` (уже стоит, сглаживает скролл) покрывают все нужные эффекты; новая тяжёлая зависимость не оправдана.

**Базис:**
- Переиспользовать `src/06_shared/ui/motion/constants.ts` (durations + ease-пресеты) — единый «спокойный, voice-first» тайминг. Новые пресеты добавлять туда же.
- Scroll-driven: где можно — нативный CSS `animation-timeline: view()`; иначе framer-motion `useScroll`/`whileInView`/IntersectionObserver. `lenis` сглаживает скролл глобально.
- **`useReducedMotion()` + CSS `@media (prefers-reduced-motion)` — обязательно для каждой анимации** (fallback = статичное конечное состояние).
- Продолжаем существующий роадмап L1–L6 (`docs/SESSION_HANDOFF.md`) по схеме «одна анимация → подтверждение → следующая».

**Список по блокам (всё утверждено в живом прототипе):**

**Reveal блоков (по входу во вьюпорт):**
- **Основной reveal — «A · rise»:** `translateY(~36px)+opacity` с ease-out, без overshoot. Применять к большинству секций.
- **Карточки — «C · scale-up»:** `scale(.9→1)+opacity`. Stagger детей в сетках (bento/сценарии/преимущества).
- **Телефон (hero) — без reveal-анимации.**
- База — видимое состояние; motion только внутри `@supports`+`@media (no-preference)`.

**Hero-заголовки (ВСЕ H1: Home/Pricing/HowItWorks) — letter-reveal:**
- Framer «blur-text»: каждая буква (`inline-block`) приходит через `blur(12px)→0 + opacity + translateY(.32em)`, stagger ~0.03s, **on-load**. Слова не рвутся. reduced-motion → текст сразу.

**Парящие элементы (badges, PainMirror-пузыри+чипы, TrustStrip-пилюли, PageHero стат-пилюли, side-карточки телефона):**
- **Прилёт** с разных сторон (`translate`+`rotate`+`scale`), приземление **сразу на свой resting-наклон** (конечный кадр прилёта = 0% кадр дрейфа, чтобы не было рывка) → затем тихий **idle-дрейф** (малая амплитуда, взаимно простые периоды).
- Триггер: **hero/выше сгиба — on-load**; ниже сгиба — при входе во вьюпорт; «8 ситуаций»-чипы могут быть scroll-linked (реверсивно).

**Hero — фон (A3):** параллакс-сетка + бледные **voice-мотивы** (мини-волны/микрофон/спич-бабл, line-иконки из shared/Icon, низкая opacity). Параллакс: десктоп — от курсора, **мобайл — от скролла** (меньшая амплитуда) либо статично. reduced-motion → статика.

**Hero — телефон:** **3D-наклон к курсору** (`rotateX/rotateY`, плавный возврат); side-карточки приподняты по `translateZ` для параллакса; **внутрянка статична**; существующий `BreathingPhone` idle сохраняется; мобайл — gyro или статично; reduced-motion → без наклона.

**8 ситуаций — iOS-wheel:** scroll-driven `scale/opacity/rotateX` относительно центра; фокусная строка — крупная/белая/тень + **хук виден (`opacity:1`), но место под хук зарезервировано всегда** (без layout-сдвига). CSS `animation-timeline: view()` или framer-motion `useScroll`. reduced-motion → статичный список.

**Переключатель цены I1:** число **пересчитывается на месте** (rAF-счётчик от текущего к целевому, ease); юнит и подпись — **мгновенно** (без fade/движения). reduced-motion → сразу финал.

**FinalCta:** волна = **единый непрерывный градиент блока**, светлая «шапка» цвета верхней секции вырезает волну (без шва, без рассинхрона цвета); voice-иконка = **Telegram-voice** (неровные бары), пульсирует; glow «дышит» из-за волны. reduced-motion → статика.

**Дропдаун языка:** меню «вырастает» из правого-верхнего угла (`scale+opacity`), пункты — stagger, каретка ▾ крупная emerald, поворот 180°. Esc/клик-вне закрывают. reduced-motion → мгновенно.

**Eyebrow-ротация (Hero):** существующий `RotatingStatus` (AnimatePresence cross-fade).

**Нижний прогрессивный блюр:** фикс-полоса у низа вьюпорта, `backdrop-filter:blur` + `mask`-градиент (сильнее у низа). **Гаснет (`opacity→0`) в пределах ~120px до конца страницы** (не мылит футер); на коротких страницах без скролла — нет. Нет `backdrop-filter` / reduced-motion → выключен.

**Hover (всё CSS-transition, `@media(hover:hover)`, `:active` для тапа, reduced-motion → мгновенно):**
- **Primary-кнопка (CTA):** заливка → **transparent (ghost)**, бордер+текст остаются (белые на emerald / зелёные на светлом). Бордер 2px всегда → без сдвига.
- **Secondary (outline):** прозрачная → **заливка primary + белый текст**.
- **Tertiary / текстовая кнопка:** подчёркивание из-под слева (scaleX, origin left).
- **Ссылки меню (хедер):** подчёркивание из-под слева (scaleX).
- **Ссылки футера:** смена цвета → primary.
- **Карточки:** подъём `-4px` + тень + emerald-рамка.
- **Строки-сценарии:** сдвиг вправо + стрелка «Открыть» отъезжает.

**Волна-divider:** одиночная плавная волна, цвет = секция-приёмник (см. §1). Анимации у самой волны нет (структурный элемент); `hero-waveform-mask` на /how-it-works — текущая анимация сохраняется.
